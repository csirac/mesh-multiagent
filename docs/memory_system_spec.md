## 1. Overview

A two-tier episodic memory system for mesh agents. After each substantial task completion, the agent reflects on what happened and produces a memory entry. A facility-location submodular function selects the most diverse entries for the active set.

**Two-tier architecture:**
- **Pool (P)** — All memories ever created, persisted in SQLite. Bounded by `pool_max_entries` (default 1000). Oldest pool-only entries are pruned when exceeded.
- **Active set (S ⊂ P)** — Top `active_size` entries (default 30) selected by submodular optimization. Displayed in the router's `<memory>` block.

Worker retrieval searches **all of P** by similarity, not just S. Workers can recall memories the router doesn't show.

The memory is injected as a `<memory>` block in the router's system prompt and in worker prompts (top-K relevant).

### Design Decisions

1. **Persistence**: Per agent **nickname** — survives router version changes. `~/.mesh/memory/{nickname}.db`
2. **User access**: Yes — exposed as mesh tools (`memory_list`, `memory_get`, `memory_delete`, `memory_add`, `memory_search`)
3. **Cross-agent sharing**: No
4. **Selection model**: Facility location submodular function with dynamic withholding-cost swapping
5. **Entry merging**: No — entries are accepted or rejected, not merged
6. **LLM-generated retrieval keys**: Task descriptors generated by the reflection LLM, not raw trigger text

## 2. Memory Entry Format — Three-Tier Structure

Each memory consists of three layers of detail:

| Tier | Content | Size | When Seen |
|------|---------|------|-----------|
| **Summary** | One paragraph: what happened + lessons distilled | ~100 tokens | Always in context (router + worker) |
| **Reflection** | Deep LLM reflection: what worked, what didn't, strategy analysis | ~500 tokens | In context for top entries + on demand via `remember(id)` |
| **Trace** | Stack trace of tool calls, function outputs (clipped) | ~2K tokens | On demand via `remember(id, full=true)` |

```python
@dataclass
class MemoryEntry:
    id: str                    # 12-char hex UUID
    created_at: datetime       # When this memory was created
    # Three-tier content
    summary: str               # Tier 1: one paragraph distillation
    reflection: str            # Tier 2: deep LLM reflection
    trace: str                 # Tier 3: clipped tool call trace
    trigger: str               # Original trigger message (provenance)
    retrieval_key: str = ""    # LLM-generated task descriptor for retrieval
    # Metadata
    tags: list[str]            # Domain tags: ["nginx", "benchmark", "mesh-routing"]
    outcome: str               # "success" | "partial" | "failure"
    # Embeddings
    reflection_embedding: np.ndarray   # For diversity selection (active set)
    retrieval_key_embedding: np.ndarray  # For task-similarity retrieval (workers)
    weight: float              # Cached withholding cost (recomputed on swaps)
```

### 2.1 What Gets Embedded

Two embeddings are computed per memory entry:

1. **Reflection embedding** — used for facility location diversity selection (accept/evict decisions). The reflection captures meta-cognitive reasoning — the right abstraction for measuring information diversity across the memory set.

2. **Retrieval key embedding** — used for worker retrieval (top-K by task similarity). The retrieval key is an LLM-generated 1-2 sentence task descriptor produced during reflection. This normalizes quality: even terse triggers like "hey bob, status?" get a specific retrieval key like "Checking status of running agents and mesh router health on sobek."

Why not embed the raw trigger or summary:
- **Raw trigger** is unreliable — many triggers are too short/generic for meaningful similarity
- **Summary** includes outcome/lesson information that dilutes task-similarity matching
- **Trace** is too noisy — tool names cause embeddings to cluster by tool usage pattern

### 2.2 Context Window Rendering

Summaries are **front-loaded** into the `<memory>` block. The agent sees them automatically. Top entries and recent entries include full reflection text. The `remember` tool lets the agent drill into any entry.

**Router rendering** (`render_block()`):
- Active set entries (summaries by default)
- Top N by weight get full reflections (`router_full_reflections`, default 0)
- Most recent N entries also get full reflections (`router_recent_reflections`, default 3)

**Worker rendering** (`render_block_for_query(query, k)`):
- Top-K from full pool by retrieval key cosine similarity
- Top N get full reflections (`worker_full_reflections`, default 2)
- Recent N entries added with full reflections (`worker_recent_reflections`, default 2)

```xml
<memory>
<entry id="a1b2c3" date="2026-02-06" tags="benchmark, mesh-routing" outcome="success" depth="full">
[a1b2c3 | 2026-02-06 14:30]
Deep reflection text here with what worked, what didn't...
</entry>
<entry id="d4e5f6" date="2026-02-05" tags="nginx, config" outcome="success" depth="summary">
One paragraph summary of what happened and key lesson learned.
</entry>
</memory>
```

### 2.3 The `remember` Tool

Both the router and worker have access to a `remember` tool:

```python
remember(id: str)              # returns reflection (~500 tokens)
remember(id: str, full=True)   # returns reflection + trace (~2K tokens)
```

### 2.4 The `memory_search` Tool

Semantic search over the full pool:

```python
memory_search(query: str, k: int = 5)  # returns top-K by retrieval key similarity
```

## 3. Memory Lifecycle

### 3.1 Reflection Gating — Three-Path Significance Filter

**When**: After `_handle_worker_complete()` succeeds (via `should_reflect()`).

**Three-path OR rule** (any path qualifying triggers reflection):

```python
should_reflect = (
    tool_heavy:    stats.tool_calls >= 3
    OR extended:   stats.num_user_visible_turns >= 4
                   AND stats.total_user_visible_chars >= 1500
    OR brainstorm: stats.tool_calls <= 2
                   AND stats.agent_response_chars >= 1500
                   AND stats.num_user_visible_turns >= 2
    OR errors:     stats.has_errors
)
```

**Additional guards:**
- **Cooldown**: 5-minute (300s) minimum between reflections to prevent storms
- **User-visible counting only**: Only user↔agent/channel messages count toward turns/chars; tool calls, tool results, and internal messages are excluded

**`EpisodeStats` dataclass:**
```python
@dataclass
class EpisodeStats:
    tool_calls: int = 0
    num_user_visible_turns: int = 0
    total_user_visible_chars: int = 0
    agent_response_chars: int = 0
    has_errors: bool = False

    def merge(self, other: "EpisodeStats") -> None:
        """Accumulate another completion's stats into this session."""
```

### 3.2 Session-Level Episode Merging

Stats are accumulated at the session level in the router. A session is a sequence of completions with no gap exceeding `memory_reflection_session_gap_secs` (default 900s / 15 minutes).

- Stats from each completion are merged into the running session stats via `EpisodeStats.merge()`
- Reflection is triggered only when session-accumulated stats meet the threshold
- On disconnect, any pending session stats are flushed for reflection

### 3.3 Reflection (Entry Generation)

**How**: A lightweight LLM call (using the agent's LLM backend) produces:
- `<reflection>`: 2-3 paragraphs of what worked, what didn't
- `<summary>`: One paragraph distillation
- `<tags>`: Comma-separated domain tags
- `<outcome_label>`: success | partial | failure
- `<retrieval_key>`: 1-2 sentence task descriptor (specific technologies/systems named)

**Fallback**: If the LLM doesn't produce `<retrieval_key>`, the summary text is used instead.

**Trace** is constructed mechanically from the worker's tool call log (no LLM needed).

Runs as `asyncio.create_task` — does not delay the user's response.

### 3.4 Two-Tier Acceptance

After reflection, the entry always goes into the **pool (P)**. Then an incremental active set update decides whether it also enters **S**:

1. **If `|S| < active_size`**: accept unconditionally, recompute weights
2. **If `|S| >= active_size`**: compute candidate's marginal gain `w_m`, compare against cached `w_min`:
   - If `w_m > w_min`: swap — evict weakest entry from S (stays in pool), insert candidate
   - If `w_m <= w_min`: reject from S (stays in pool only)

### 3.5 Pool Pruning

When pool exceeds `pool_max_entries`:
- Only **pool-only entries** (not in S) are candidates for pruning
- Sorted by `created_at` ascending (oldest first)
- Active entries are never pruned

### 3.6 Facility Location Selection

Uses a facility location submodular function `f` for all selection decisions:

```
f(S) = Σ_{v ∈ S} max_{s ∈ S} sim(v, s)
```

This measures the total "coverage" of the set — each entry's value is determined by how well it's represented by its nearest neighbor. A diverse set has high `f(S)` because entries cover distinct regions of embedding space.

#### The Swapping Algorithm

All decisions use `f`. Withholding costs are **cached** and only recomputed when the memory set changes (i.e., a swap occurs).

**On startup / after any swap (O(n²)):**
- For every `a ∈ S`, compute and cache the withholding cost:
  `w_a = f(S) - f(S \ {a})`
- Cache `w_min = min(w_a)` and its index for fast lookup

**On candidate `m` arrival:**
1. **Embed** the candidate reflection
2. **If memory is not full** (`|S| < max_entries`): accept unconditionally, then recompute all cached weights (O(n²))
3. **If memory is full**: compute candidate marginal gain using the same `f`:
   `w_m = f(S ∪ {m}) - f(S)` — O(n)
4. Compare `w_m` against cached `w_min` — O(1):
   - **If `w_m > w_min`**: **swap** — evict `argmin_a w_a`, insert `m`, recompute all cached withholding costs (O(n²))
   - **If `w_m <= w_min`**: **reject** — done. No recomputation needed.

**Cold start**: When `|S| < max_entries`, all candidates are accepted unconditionally (step 2). Withholding costs are recomputed after each addition so cached weights are always current when the store reaches capacity.

**Key properties:**
- **Same function everywhere**: `w_m` and `w_a` are both marginal contributions under the same `f`. The comparison `w_m > min w_a` has clean semantics: "the candidate contributes more to set diversity than the weakest existing entry."
- **Rejection is cheap**: O(n) — one marginal gain computation + comparison against cached minimum. This is the common case as memory matures.
- **Acceptance triggers recomputation**: O(n²) — all withholding costs must be recomputed in the context of the new set. Increasingly rare as memory fills and stabilizes.
- **Dynamic revaluation**: Withholding costs reflect the current set composition. A memory that was high-value when inserted but is now redundant (because similar memories were added since) will have a low withholding cost after recomputation and become a natural swap candidate.

#### Implementation

```python
class MemoryStore:
    def __init__(self, max_entries: int):
        self.max_entries = max_entries
        self.entries: list[MemoryEntry] = []
        self.cached_weights: list[float] = []  # withholding costs
        self.cached_min_idx: int = -1
        self.cached_min_weight: float = float('inf')

    def _recompute_weights(self):
        """O(n²) — recompute all withholding costs."""
        all_embs = [m.reflection_embedding for m in self.entries]
        baseline = facility_location(all_embs)
        self.cached_weights = []
        for i in range(len(self.entries)):
            s_without_i = all_embs[:i] + all_embs[i+1:]
            w_i = baseline - facility_location(s_without_i)
            self.cached_weights.append(w_i)
        if self.cached_weights:
            self.cached_min_idx = int(np.argmin(self.cached_weights))
            self.cached_min_weight = self.cached_weights[self.cached_min_idx]

    def try_insert(self, candidate: MemoryEntry) -> tuple[bool, int | None]:
        """
        O(n) if rejected (common case), O(n²) if accepted (triggers recomputation).
        Returns (accepted, evicted_index).
        """
        # Cold start: accept unconditionally until full
        if len(self.entries) < self.max_entries:
            return True, None  # caller adds entry, then calls _recompute_weights

        # O(n): compute candidate's marginal gain under f
        all_embs = [m.reflection_embedding for m in self.entries]
        candidate_emb = candidate.reflection_embedding
        w_m = facility_location(all_embs + [candidate_emb]) - facility_location(all_embs)

        # Compare against cached minimum withholding cost — O(1)
        if w_m > self.cached_min_weight:
            evict_idx = self.cached_min_idx
            return True, evict_idx  # caller does swap, then calls _recompute_weights
        return False, None  # rejected — no recomputation needed


def facility_location(embs: list[np.ndarray]) -> float:
    """Facility location: total coverage of the embedding set."""
    if not embs:
        return 0.0
    total = 0.0
    for v in embs:
        total += max(cosine_sim(v, s) for s in embs)
    return total
```

#### Generalizing Later

The submodular function can be extended to incorporate additional signals beyond embedding similarity. For example, access frequency, recency, or domain-specific relevance weights could be folded into the `s_ij` similarity function or added as multiplicative factors on the withholding cost. This keeps the same algorithmic framework while enriching the notion of "value."

### 3.3 Retrieval (Injecting into Prompts)

**For the router prompt** (`_build_router_prompt`):
- Include **all** memory entries (fixed-size, bounded)
- Placed in `<memory>` block after `<identity>`, before `<tools>`
- Full experiential context for classification

**For worker prompts** (via `_build_system_prompt_with_preferences`):
- Retrieve **top-K most relevant** entries by cosine similarity between the incoming trigger embedding and stored **retrieval key embeddings**
- This keeps retrieval in the same semantic space: "past tasks similar to this task"
- K defaults to 5 (configurable)
- Router curates what each worker sees

## 4. Storage Backend

### 4.1 SQLite Schema

Database at `~/.mesh/memory/{nickname}.db`:

```sql
CREATE TABLE memories (
    id TEXT PRIMARY KEY,
    created_at TEXT NOT NULL,
    summary TEXT NOT NULL,
    reflection TEXT NOT NULL,
    trace TEXT NOT NULL,
    trigger_text TEXT NOT NULL,
    retrieval_key TEXT NOT NULL DEFAULT '',
    tags TEXT NOT NULL,
    outcome TEXT NOT NULL,
    reflection_embedding BLOB NOT NULL,
    retrieval_key_embedding BLOB NOT NULL,
    weight REAL NOT NULL DEFAULT 0.0
);

CREATE TABLE meta (
    key TEXT PRIMARY KEY,
    value TEXT
);
```

**Migration**: On open, `_migrate()` detects old schema (`trigger_embedding` column) and renames it to `retrieval_key_embedding`, adding `retrieval_key` column if missing. Idempotent.

### 4.2 Embedding Model

Default: OpenAI `text-embedding-3-small` (1536 dims, $0.02/1M tokens).

Configurable via `memory_embedding_backend` and `memory_embedding_model` in config.

## 8. Scripts

### 8.1 Backfill Script (`mesh/scripts/backfill_memory.py`)

Populates a memory pool from an agent's full conversation history:

```bash
python -m mesh.scripts.backfill_memory --agent sysadmin:bob --nickname bob
```

Options:
- `--dry-run` — Show candidates without writing
- `--backend` / `--model` — Override reflection LLM (default: openai / gpt-5.1)

Parses history into session-merged episodes (same 15-minute gap logic as the online path), applies the three-path significance filter, generates reflections, and writes to the memory DB.

### 8.2 Retrofit Script (`mesh/scripts/retrofit_retrieval_keys.py`)

Generates LLM-generated retrieval keys for existing memories that have empty keys:

```bash
python -m mesh.scripts.retrofit_retrieval_keys --nickname bob
```

Options:
- `--dry-run` — Show what would be updated
- `--backend` / `--model` — Override LLM for key generation

Uses batch embedding (1 API call per agent) for efficiency.
